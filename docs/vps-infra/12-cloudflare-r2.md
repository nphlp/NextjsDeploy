# Cloudflare R2 — Database Backups

Cloudflare R2 is an S3-compatible object storage used for automated database backups via Dokploy.

## Prerequisites

- A [Cloudflare](https://dash.cloudflare.com) account with billing information (required to access R2)

## 1. Create an R2 Bucket

1. Go to [Cloudflare Dashboard](https://dash.cloudflare.com) → **R2 Object Storage**
2. Click **Create bucket**
3. Name: `dokploy-backups`
4. Location: **Europe de l'Ouest (WEUR)** (or closest to your VPS)
5. Leave defaults, create

## 2. Create an R2 API Token

1. In R2, go to **Account Details** → **{} Manage** (API Tokens)
2. Click **Create Account Token** (not "User Token" — Account tokens persist even if you leave the organization)
3. Settings:
    - **Name**: `R2 Account Token for Dokploy`
    - **Permissions**: `Object Read & Write` (not admin)
    - **Buckets**: Apply to specific bucket only → `dokploy-backups`
    - **TTL**: Unlimited
4. Create and **save the credentials** (shown only once):
    - **Access Key ID** (e.g. `a1b2c3d4...e5f6g7h8`) — needed for Dokploy
    - **Secret Access Key** (e.g. `a1b2c3d4...e5f6g7h8i9j0`) — needed for Dokploy
    - **Endpoint URL** (e.g. `https://<account-id>.r2.cloudflarestorage.com`) — needed for Dokploy
    - **Token Value** (e.g. `cfat_abc1...xyz9`) — not needed for S3 backups (Cloudflare REST API only)

## 3. Configure S3 Destination in Dokploy

1. Go to **Dokploy** → **Settings** → **S3 Destinations** → **Add Destination**
2. Fill in:
    - **Name**: `Dokploy Backups with Cloudflare R2`
    - **Provider**: `Cloudflare R2 Storage`
    - **Access Key Id**: from step 2
    - **Secret Access Key**: from step 2
    - **Bucket**: `dokploy-backups`
    - **Region**: `WEUR` (bucket location code, found in R2 → bucket → **Settings** tab)
    - **Endpoint**: `https://<account-id>.r2.cloudflarestorage.com`
3. Click **Test connection** to verify
4. Save

## 4. Configure Backups

### Dokploy Internal Database

1. Go to **Dokploy** → **Settings** → **Server**
2. Add S3 backup:
    - **Destination**: `Dokploy Backups with Cloudflare R2`
    - **Database**: `dokploy` (default Dokploy database name)
    - **Schedule**: `Every Sunday at midnight` (`0 0 * * 0`)
    - **Prefix Destination**: `/`
    - **Keep latest**: `3`
    - **Enabled**: yes
3. Run a manual backup to verify

### Project Databases

For each standalone database in your projects:

1. Go to **Projects** → your project → your environment → your database service
2. Go to the **Backups** tab → **Create Backup**
3. Fill in:
    - **Destination**: `Dokploy Backups with Cloudflare R2`
    - **Database**: the database name (found in the **General** tab of the service)
    - **Schedule**: `Every Sunday at midnight` (`0 0 * * 0`)
    - **Prefix Destination**: `/`
    - **Keep latest**: `3`
    - **Enabled**: yes
4. Run a manual backup to verify

## 5. Verify

Go to [Cloudflare Dashboard](https://dash.cloudflare.com) → **R2** → `dokploy-backups`. You should see one directory per backup source:

You should see one directory per backup source (names are auto-generated by Dokploy).

## Pricing

Cloudflare R2 includes a generous free tier:

- **10 GB** storage free
- **10 million** Class A operations (writes) free per month
- **1 million** Class B operations (reads) free per month
- No egress fees

For database backups, you'll likely stay within the free tier.
